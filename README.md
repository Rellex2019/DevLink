# DevLink - совместное кодироапние

## Описание  
Проект представляет собой каркас - основной функционал для реализации простой платформы для совместого кодинга.  

Тут вы врядли найдете пример best practic, т.к проект писался для изучения различных технологий. 
(Где-то нет необходимых посредников, могут быть различные тестовые секреты и т.д)

### Реализованные системы: 
#### 1. Репозитории:
- Просмотр репозиториев (просмотр - открытый репозиторий, редактирование - есть доступ у команды, либо владелец);
- Создание репозиториев;
- Настройка репозиториев;
- Приглашение команд;
- Создание файлов и папок (простая реализация, через сохранение в виде записей в БД, а не локально файлами);
- Поиск среди файлов;

#### 2. Команды:
- Создание команды;
- Приглашение участников;
- Настройки команды;
- Просмотр доступных репозиториев (там где было принято приглашение);

#### 3. Kanban-доска задач (доступка в разделе Репозиторий->Задачи, после принятия приглашения от владельца репозитория, командой):
- Создание, удаление статусов;
- Создание, перемещение, удаление задач;
- Просмостр задач;
- Создание комментария к задаче(функционал был вырезан, но фронтовая реализация осталась);
- Поиск задачи;

#### 4. Глобальный поиск
Был реализован в старых коммитах при помощи полнотекстового поиска ElasticSearch (изучал технологию), но был заменен на обычный поиск по ключевым словам.

## Установка  
- Необходимо установить Docker.
- Необходимо установить старое состояние репозитория до раздробление на сервисы, для наибольшей работоспособности.

1. Клонируйте репозиторий:  
   ```bash  
   git clone https://github.com/Rellex2019/DevLink.git
   ```
   ```bash
   cd DevLink
   ```
2. Откатить изменения до коммита "Итоговый проект"
   ```bash
   git reset --hard 2d528d3
   ```
3. Запутить сборку Docker
    #### Здесь используется много сервисов, которые можно перенести на другие сервера изменив docker-compose.yml
    ```docker
    docker-compose up -d --build
    ```
4. Запустить миграции при необходимости
    ```docker
    docker-compose exec app php artisan migrate
    ```
5. Готово
    #### Проект доступен по адресу [http://localhost:80](http://localhost/)
